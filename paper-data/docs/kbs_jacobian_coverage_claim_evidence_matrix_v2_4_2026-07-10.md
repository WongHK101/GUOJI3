# Jacobian Coverage Audit Manuscript: Claim-Evidence Matrix v2.4

## Scope and terminology ledger

This matrix governs the separate submission candidate `istf_kbs_jacobian_coverage_submission_candidate_v2_4.tex`. It does not replace the canonical KBS manuscript. All empirical entries are frozen artifacts. P1 did not inspect or use Stage 1b model-training or model-performance outputs for seeds 4-8. V2.4 uses read-only source inspection and frozen-data document/plot transforms only; it adds no training, GPU run, Stage 1b work, or new empirical result.

| Canonical term | Definition used in v2.4 | Do not substitute with |
| --- | --- | --- |
| directed Granger-predictive dependency graph | The graph induced by an explicitly stated predictive derivative and aggregation rule. | General causal graph, unless stronger assumptions are stated. |
| Granger graph knowledge | The reported directed Granger-predictive dependency object. | Causal knowledge without qualification. |
| Jacobian coverage audit | A claim-specific audit of score-route completeness, penalty-route completeness, score-penalty alignment, coordinate validity, and horizon validity. | Coverage certificate or identifiability proof. |
| route ledger | `P_pred={(r,rho_score(r),rho_penalty(r),exempt(r))}` for architecture-declared predictive route classes, with score, penalty, and exemption status recorded independently. | Enumeration of every activated neural path or mutually exclusive route categories. |
| audit profile | The set of all applicable coverage, coordinate, horizon, and unassessed flags for one declared graph-score claim. | A single precedence-ordered label. |
| CLAIM-COVERED | A profile outcome used only when all applicable declared dimensions pass and no required status is unknown. | Model-wide guarantee or a positive label inferred from incomplete evidence. |

## Source-verified controlled concat instantiation

| Question | Verified answer | Source |
| --- | --- | --- |
| Is `c` external? | No. In the controlled diagnostics, `c_u=g_phi(X_{0:u})` is a learned deterministic transform of the same raw sequence. | `src/mamba_jrngc_pilot.py::MambaPreprocessor.forward` and `MambaJRNGC.preprocess_and_windowing`. |
| What is `g_phi`? | A pure-PyTorch causal selective state-space block with LayerNorm, cross-channel projections, SiLU, causal prefix recurrence and residual path, followed by a linear `d -> d_cond` head. Controlled settings use `d_state=4`, expansion 2, projection width 4 and residual gain 0.1. | `src/minimal_mamba.py::SelectiveSSM`, `MambaBlock`; controlled experiment scripts. |
| What temporal support is used? | `c_u` can depend on the full raw prefix through `u`; filter state is not reset at the lag-window boundary. The predictor for raw target `x_t` receives only `c_{t-K:t-1}`. | `_parallel_scan`, `preprocess_and_windowing`, `compute_loss`. |
| What enters the predictor? | Flattened concatenation `[X_{t-K:t-1}; c_{t-K:t-1}]` with `(d+d_cond)K` coordinates; target is raw `x_t`. | `MambaJRNGC.compute_loss`. |
| Where is detach used? | Prediction uses no detach through `g_phi`. Scoring/penalty detach both stored blocks from preprocessing, re-enable gradients only for raw `X` in the x-only implementation, and hold `c` fixed. | `compute_jacobian_loss`, `get_gc_matrix`. |
| What does the full penalty change? | It enables gradients for all concatenated `x` and `c` coordinates and applies separate mean-absolute penalties. It does not override the inherited x-only graph extractor. | `experiments/risk_mitigation_20260515/run_full_aux_penalty.py::MambaConcatFullPenaltyJRNGC`. |

The architecture row does not generalize to an external covariate, a conditional graph over `X|c`, or a joint graph over `(X,c)`. Those are distinct declarations. No verified architecture item remains unresolved for the controlled concat instantiation.
