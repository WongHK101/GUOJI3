# Route-B Draft v2.3 Science-Lock Changelog

## Scope

V2.3 is a tightly bounded science-lock revision of the accepted v2.2 draft.
It uses frozen evidence only. It does not replace canonical `istf_kbs.tex`,
run an experiment, train a model, use a GPU, start Stage 1b, or inspect Stage
1b model-training or model-performance outputs.

## V2.2 to v2.3 corrections

1. Added an objective-level formal remark immediately after Proposition 3.1.
   Under the proposition's assumptions, the auxiliary-only construction has
   `R_x=0`, so a low x-only regularized objective does not itself certify that
   the x-only Jacobian score represents information used for prediction. The
   remark explicitly excludes optimizer-selection, empirical-degradation, and
   universal-conditioned-predictor claims.
2. Replaced the four-dimension formulation with five distinct dimensions:
   score-route completeness, penalty-route completeness, score-penalty
   alignment, coordinate validity, and horizon validity.
3. Replaced the single-label audit output with a claim-specific audit profile.
   `CLAIM-COVERED` now requires all applicable dimensions to pass with no
   required unknown status. All applicable failure and unassessed flags are
   reported, and `PARTIALLY COVERED` may coexist with coordinate or horizon
   flags.
4. Corrected route-ledger wording so score, penalty, and exemption status are
   independently recorded. A route may be both scored and penalized.
5. Updated Abstract, Introduction, contribution list, Section 4, Algorithm 1,
   Figure 1 and Figure 3 captions, Discussion, Conclusion, Table 1 wording,
   and the appendix audit template to use the same five-dimension profile
   semantics. Table 1 retains its v2.2 five-column geometry.
6. Replaced the broad P1 seeds statement with the exact bounded wording:
   `P1 did not inspect or use Stage 1b model-training or model-performance
   outputs for seeds 4--8.` The same wording is used in the main text,
   appendix, claim-evidence matrix, traceability, and evidence index.
7. Regenerated conceptual figure labels with the existing Python source and
   frozen evidence. Figure 1 now shows independently recorded route statuses
   and coexisting profile flags. Figure 3 names coordinate validity and horizon
   validity separately. No empirical value, error bar, or data source changed.
8. Added a standalone v2.3 audit-report template with separate determinations
   for all five dimensions and a set-valued audit profile.
